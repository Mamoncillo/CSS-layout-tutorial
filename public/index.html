display: block vs inline

css info

Display property

The display: CSS property specifies the type of rendering box used for an element

There are two generic tags that have block and inline values by default

Block Inline
<div> <span>

in addition there are also a bunch of semantic tags with this values by default too

Block Inline
<h1>, <h2> <a>
<p> <img>
<ul>, <ol> <input>
<table> <textarea>
<blockquote> <code>
<video> <strong>
... ...

Always try to use the semantic tags, but if you think that no one of them makes sense, then use one of the generic tags

Block

in block means that the element will take all the line and will put the other elements below him

Inline

in line means that the element will take all the line but instead of put the other elements below him, he will share the line with other inline elements

You can't set width or height of inline elements, but you can use inline-block instead

none

The element will be not displayed
